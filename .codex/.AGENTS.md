# Agent Instructions

## Collaboration

- Always ask the user clarifying questions when requirements, intent, or implementation details are uncertain.
- Always provide the user with a plan before making code changes.
- Do not proceed with code changes until the plan has been shared.

## Engineering Standards

- When coding, add or update appropriate unit tests for the change.
- Run the relevant test suite before considering the work complete.
- Ensure all tests pass before proceeding to the next implementation step or reporting completion.
- If tests cannot be run, clearly explain why and identify the remaining risk.

## Security

- Never commit private keys, secrets, tokens, certificates, credentials, or production connection strings.
- Use secure local or cloud secret storage for sensitive values.

## Pull Requests

- Pull requests may be raised when useful.
- Never merge changes into the `main` branch.
- Before beginning a new feature, create a new branch from `main`.
- If unsure whether a new branch should be created, ask the user before proceeding.

## Project Context

- This project is a react native app using Expo which enables users to track pushups exercises over a period of time.
- There is an intention to monetise via a subscription.
- The app is built for iOS and Android and leverages the device sensors to track pushups.
